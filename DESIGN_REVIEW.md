# CheckMate UX/UI 디자인 리뷰 및 보완점

이 문서는 Figma에 정의된 4개의 주요 페이지(메인 분석 대시보드, 분석 결과 상세 페이지, 신뢰도 통계 및 히스토리, About CheckMate)에 대한 UX/UI 리뷰 및 프론트엔드 개발 시 고려해야 할 보완 사항을 기록합니다.

## 1. 인터랙션 및 상태(상태 피드백) 설계 보완
*   **로딩(Loading) 상태 누락**:
    *   메인 페이지에서 URL을 입력하고 '분석' 버튼을 눌렀을 때, 분석 결과가 나오기까지(특히 AI 분석의 경우 시간이 소요됨) 화면의 변화 상태 정의가 필요합니다.
    *   **해결 방안**: 스켈레톤(Skeleton) UI, 프로세스 프로그레스 바(Progress Bar), 혹은 부드러운 로딩 스피너 애니메이션을 적용해야 합니다.
*   **폼(Form) 요소의 Focus/Error 상태**:
    *   URL 입력창을 클릭했을 때(Focus) 시각적 강조 효과 적용.
    *   잘못된 형태의 URL(뉴스 링크가 아님) 입력 시 명확한 에러 메시지(Red text)와 컴포넌트 오류 테두리 표시 규칙이 필요합니다.

## 2. 접근성(Accessibility) 및 명도 대비 확인
*   **가독성(Contrast)**:
    *   신뢰도 점수(Trust Score Gauge)를 표시하는 그라데이션 배경 위의 텍스트나, 카드 UI의 서브 텍스트(예: 작성일, 작성자 등 밝은 회색 폰트)가 배경색과 충분한 명도 대비(최소 4.5:1)를 가지는지 점검이 필요합니다.
    *   실제 디스플레이 환경을 고려해 서브 텍스트의 대비를 조금 더 높이는(진하게) 것이 좋습니다.

## 3. 차트 및 데이터 시각화 디테일 (신뢰도 통계/히스토리 페이지)
*   **차트 툴팁(Tooltip) 추가**:
    *   `신뢰도 통계 및 히스토리` 화면의 꺾은선/바 차트에서 특정 지점을 마우스 호버(Hover)하거나 터치했을 때, 해당 날짜의 기사 제목과 점수를 알려주는 툴팁 UI가 추가되면 데이터를 이해하기 좋습니다.
*   **색상의 의미 부여(Semantic Colors)**:
    *   신뢰도가 높은 기사, 주의가 필요한 기사, 허위/위험 기사에 대해 일관된 색상 토큰(예: 높음=Blue/Green, 낮음=Red/Orange)을 고정하여 사용자에게 직관적인 정보를 제공해야 합니다.

## 4. 반응형 웹(Responsive Web) 고려 (모바일 대응)
*   **Bento Grid의 모바일 레이아웃**:
    *   현재 데스크탑(1280px) 모드 기준의 다단(Multi-column) Bento Grid 레이아웃입니다.
    *   모바일 뷰포트(가로 폭 375px~430px)로 좁아질 때, 카드들이 1열(1 Column)로 재배치될 때의 여백(Spacing)과 순서를 정의해야 합니다.
*   **텍스트 말줄임(Truncation) 처리**:
    *   모바일 환경에서 긴 기사 제목이나 내용이 레이아웃을 깨지 않도록 `text-overflow: ellipsis` 혹은 `line-clamp` 속성을 활용해 1~2줄로 제한하는 규칙을 미리 설정해야 합니다.
